features

Everything you need.
Nothing you don't.

No cloud accounts. No subscriptions. Runs entirely on your machine with a scheduler that installs itself.

snap

Timestamped snapshots

Every backup run creates a new dated snapshot — not an overwrite. You decide how many to keep, old ones are cleaned up automatically. Every version is there if you need it.

sched

Runs on a schedule, automatically

Set a backup interval per job (e.g. every 2 hours). zipp installs a system scheduler — launchd on macOS, systemd on Linux — so it keeps running even when you close the app. No cron, no scripts.

.gz

Compressed archives or folder copy

Toggle compression per job. Compressed mode packs everything into a .tar.gz — good for large game worlds or databases. Folder mode keeps files as-is — good for quick access.

mode

Local, remote, or both

Press n on any job to switch between three modes. [local] — backup stays on this machine. [nest] — backup goes to your remote server only. [nest+local] — both at once.

rest

One-key restore

Press r on any job to see all its snapshots listed by date. Pick one, confirm — zipp puts the files back exactly where they came from. Done.

zipp-nest

A self-hosted backup server that pairs with zipp.
Install it on any machine — your home server, a VPS, a Raspberry Pi.
Backups are end-to-end encrypted. Your data never touches anyone else's server.

Install on server GitHub ↗

what you get

priv

Private by design

Backups travel end-to-end encrypted between your machines. No open ports, no S3 bucket, no third-party storage. Works on a local network or remotely — just two machines talking directly to each other.

logs

Built-in logs

Every upload is written to ~/.zipp-nest/server.log with timestamp, job name and size. View the full history from the TUI Logs page without leaving the terminal.

prune

Server-side auto-prune

After each upload zipp sends a prune request to the server. Old snapshots are deleted to match your max snapshots setting — no manual cleanup needed on either side.

cfg

Configurable via TUI

Change the storage path or port from the Settings page. If the server is running it restarts automatically — no config files to edit by hand.

setup

Four steps to remote backups.

1

Install zipp on your client machine

One curl command. Binary in /usr/local/bin, scheduler registered automatically.

2

Install zipp-nest on your server

Same process. Open the TUI, press Start server — it registers as a background service.

3

Connect to your server

zipp-nest shows a short connection code — paste it into zipp on the Nest page. Works over local network or remotely.

4

Set mode and run

Press n on any job to pick [local], [nest], or [nest+local]. Backups go where you want them.
