what you get

Everything you need.
Nothing you don't.

zipp runs entirely on your machine. No cloud accounts, no subscriptions, no background services you didn't ask for.

📸

Snapshots

Every backup creates a timestamped snapshot. Keep as many as you want, auto-prune the rest.

⏱

Scheduled

Set an interval per job. zipp installs a launchd / systemd timer — no cron wrangling.

🗜

Compressed

Toggle [zip] mode per job to store archives instead of folders. Saves significant space.

↩

One-click restore

Browse snapshots and restore any of them directly from the TUI. No extra tools needed.

📡

Remote ready

Pair with zipp-nest to push backups over Tailscale to any machine on your private network.

🖥

macOS & Linux

Single binary, installs in seconds. Works on your laptop and on your server.

installation

One line install.

introducing

zipp-nest

Your backups, over the internet.
A self-hosted backup server that receives uploads from zipp
and stores them on any machine you control.

Install on server GitHub ↗
🔒

Private by design

Runs over Tailscale — your backups never touch a third-party server. Your network, your rules.

🗂

Per-job storage

Each job gets its own folder on the server. Browse the log from the TUI to see every upload.

✂

Auto-prune

When zipp uploads a new snapshot, it automatically removes old ones on the server — matching your local max snapshots setting.

⚙

Configurable

Set the storage path and port from the TUI. Changes apply immediately — the server restarts automatically.

🔄

Three backup modes

Per job: [local] only, [nest] only (remote, saves local disk), or [nest+local] for both.

📋

Built-in logs

The server logs every upload with timestamp and size. View the full history without leaving the TUI.

how it works

Simple setup, no config files.

1

Install zipp on your client machine

One curl command, binary drops into /usr/local/bin.

2

Install zipp-nest on your server

Same process. Run zipp-nest and press Start server — it registers as a background service.

3

Connect with Tailscale

Enable Tailscale on both machines. zipp-nest shows you a short connection code — paste it into zipp on the client.

4

Set your backup mode and run

Press n on any job to cycle through [local] [nest] [nest+local]. That's it.
