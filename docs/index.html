zipp
simple, fast backups.

Snapshots. Scheduling. Restore.
A terminal app that stays out of your way.

Get started GitHub ↗

what you get

Everything you need.
Nothing you don't.

Runs entirely on your machine. No cloud accounts, no subscriptions, no background services you didn't ask for.

📸

Snapshots

Every backup creates a timestamped snapshot. Keep as many as you want, auto-prune the rest.

⏱

Scheduled

Set an interval per job. zipp installs a launchd / systemd timer — no cron wrangling.

🗜

Compressed

Toggle [zip] mode per job to store archives. Saves significant disk space.

↩

One-click restore

Browse snapshots and restore directly from the TUI. No extra tools needed.

📡

Remote ready

Pair with zipp-nest to push backups over Tailscale to any machine you control.

🖥

macOS & Linux

Single binary, installs in seconds. Works on your laptop and your server.

installation

One line install.

Downloads a prebuilt binary for your OS and architecture, places it in /usr/local/bin.

self-hosted
introducing

zipp-nest

Your backups, over the internet.
A self-hosted backup server that receives uploads from zipp and stores them on any machine you control.

Install on server GitHub ↗
🔒

Private by design

Runs over Tailscale — your backups never touch a third-party server. Your network, your rules.

🗂

Per-job storage

Each job gets its own folder. Browse the full upload history from the built-in Logs page.

✂

Auto-prune

After each upload, old snapshots are removed on the server to match your local max snapshots setting.

⚙

Configurable

Set storage path and port from the TUI. Changes apply immediately — the server restarts automatically.

🔄

Three backup modes

Per job: [local] only, [nest] only (saves local disk), or [nest+local] for both.

📋

Built-in logs

Every upload is logged with timestamp and size. View the full history without leaving the terminal.

how it works

Simple setup, no config files.

1

Install zipp on your client machine

One curl command, binary drops into /usr/local/bin.

2

Install zipp-nest on your server

Same process. Open the TUI, press Start server — registers as a background service automatically.

3

Connect with Tailscale

Enable Tailscale on both machines. zipp-nest shows a short code — paste it into zipp on the client.

4

Set mode and run

Press n on any job to cycle [local] → [nest] → [nest+local]. Done.
