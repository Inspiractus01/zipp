features

Everything you need.
Nothing you don't.

No cloud accounts. No subscriptions. Runs entirely on your machine with a scheduler that installs itself.

snap

Timestamped snapshots

Every run creates a new snapshot. Set how many to keep — old ones are pruned automatically. Browse and restore any of them from the TUI.

sched

Per-job scheduling

Set an interval in hours per job. zipp registers a launchd agent (macOS) or systemd timer (Linux) — runs in the background, no cron needed.

.gz

Compressed or folder mode

Toggle [zip] per job to store .tar.gz archives instead of rsync folders. Significant space savings for large directories.

mode

Three backup modes

Press n on any job to cycle: [local] keeps backups on this machine only, [nest] sends to the remote server only, [nest+local] does both.

rest

One-key restore

Press r on any job to see its snapshots. Select one, confirm — files are restored to the original source path.

what you get

priv

Private by design

Connects over Tailscale — a private encrypted network you control. No open ports, no S3 bucket, no third-party storage. Just two machines talking to each other.

logs

Built-in logs

Every upload is written to ~/.zipp-nest/server.log with timestamp, job name and size. View the full history from the TUI Logs page without leaving the terminal.

prune

Server-side auto-prune

After each upload zipp sends a prune request to the server. Old snapshots are deleted to match your max snapshots setting — no manual cleanup needed on either side.

cfg

Configurable via TUI

Change the storage path or port from the Settings page. If the server is running it restarts automatically — no config files to edit by hand.

setup

Four steps to remote backups.

1

Install zipp on your client machine

One curl command. Binary in /usr/local/bin, scheduler registered automatically.

2

Install zipp-nest on your server

Same process. Open the TUI, press Start server — it registers as a background service.

3

Connect with Tailscale

Enable Tailscale on both machines. zipp-nest shows a short connection code — paste it into zipp on the Nest page.

4

Set mode and run

Press n on any job to pick [local], [nest], or [nest+local]. Backups go where you want them.
